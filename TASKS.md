# План задач

Документ фиксирует первые задачи для двух направлений разработки.  
Формат веток:

- `task_1XX` — серверная часть;
- `task_2XX` — клиентская часть.

## Серверная часть

| Ветка | Название | Что сделать |
|---|---|---|
| `task_101` | Каркас серверной части | Проверить структуру FastAPI-проекта, настройки окружения, подключение миграций, healthcheck, базовую конфигурацию и локальный запуск. |
| `task_102` | Модуль умных ответов | Подготовить слой обработки сообщения: входящий текст, история диалога, поиск в памяти, сбор промпта, вызов провайдера модели, возврат ответа и решения об автоответе. |
| `task_103` | Регистрация, вход и профиль | Реализовать регистрацию компании и владельца, вход по почте и паролю, обновление токена, JWT access/refresh, `GET /users/me`. |
| `task_104` | Единый контракт запросов | Описать Pydantic-схемы ответов и ошибок, убрать нестрогие `dict` там, где нужен стабильный контракт, привести OpenAPI к виду, удобному для фронта. |
| `task_105` | Демонстрационные данные | Добавить начальные данные для разработки: компания, пользователь, каналы, клиенты, диалоги, сообщения, документы базы знаний. |
| `task_106` | Настройки компании и ИИ | Реализовать запросы для AI-настроек, профиля компании, тарифа и лимитов: получение и сохранение основных настроек. |
| `task_107` | Каналы обращений | Реализовать список каналов, подключение веб-чата, хранение настроек канала, базовые webhook-заготовки для внешних каналов. |
| `task_108` | Диалоги и сообщения | Реализовать список диалогов, карточку диалога, сообщения, отправку ответа менеджера, эскалацию и базовые фильтры inbox. |
| `task_109` | База знаний | Реализовать список документов, загрузку документа, просмотр статуса обработки, playground-вопрос, кандидатов автообучения. |
| `task_110` | Аналитика и проверка качества | Реализовать обзорные метрики, счётчики использования, базовые тесты для ключевых запросов и проверку стабильности серверной части. |

## Клиентская часть

| Ветка | Название | Что сделать |
|---|---|---|
| `task_201` | Каркас интерфейса | Собрать общий layout, верхнюю навигацию, публичную и защищённую зоны, базовые состояния страниц и общий визуальный стиль. |
| `task_202` | Страницы входа и регистрации | Сделать формы входа и регистрации, обработку ошибок, сохранение токенов, выход из аккаунта и переходы после авторизации. |
| `task_203` | Связь с сервером | Подключить axios-клиент, генерацию типов из OpenAPI, подстановку токена, обновление сессии и общую обработку ошибок. |
| `task_204` | Экран диалогов | Сделать inbox: список диалогов, выбранный диалог, сообщения, карточку клиента, черновик ИИ, отправку ответа и эскалацию. |
| `task_205` | Экран базы знаний | Сделать список документов, загрузку, статусы обработки, playground-вопрос, показ источников ответа и очередь кандидатов. |
| `task_206` | Экран каналов | Сделать список подключённых каналов, подключение веб-чата, статусы, ошибки и инструкцию для вставки виджета. |
| `task_207` | Настройки и профиль | Сделать настройки ИИ, профиль компании, тариф и лимиты, профиль пользователя и базовый список команды. |
| `task_208` | Экран аналитики | Сделать карточки ключевых метрик, использование лимитов, простые графики и состояния пустых данных. |
| `task_209` | Компоненты интерфейса | Разбить страницы на переиспользуемые компоненты, привести кнопки, карточки, бейджи, формы и таблицы к единому виду. |
| `task_210` | Проверка и полировка | Добавить проверки основных сценариев, состояния загрузки и ошибок, адаптивность, финальные правки интерфейса. |

## Рекомендуемый порядок

1. `task_101` и `task_201` — базовые каркасы.
2. `task_102` — задел для умных ответов.
3. `task_103` и `task_203` — настоящая авторизация и связь фронта с сервером.
4. `task_108` и `task_204` — главный рабочий экран диалогов.
5. `task_109` и `task_205` — база знаний.
6. Остальные задачи закрывают каналы, настройки, аналитику и качество.
